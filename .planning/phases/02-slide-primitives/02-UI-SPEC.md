# Phase 2 — UI Design Contract (Slide Primitives)

> Visual contract for slide content rendered via Office.js. This phase produces ON-SLIDE elements (text shapes, tables, layout regions, image placeholders), NOT taskpane UI. The taskpane is limited to test trigger buttons (minimal, at Claude's discretion per D-08). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Office.js PowerPointApi 1.8 (shapes, tables, text formatting) |
| Rendering target | PowerPoint slide canvas (NOT browser DOM) |
| Coordinate system | Points (1 pt = 1/72 inch). All positioning values in this spec are in points. |
| Slide dimensions (widescreen) | 960 x 540 pt (13.33" x 7.5") |
| Slide dimensions (standard) | 720 x 540 pt (10" x 7.5") |
| Dimension detection | Read `presentation.slideMasters` or use default 960x540; adaptive per D-01 |
| Taskpane components | Fluent UI v9 (buttons only for test triggers — minimal, per D-08) |

**Note:** All visual specifications below apply to shapes and text rendered on the PowerPoint slide canvas via Office.js API calls (`addTextBox`, `addTable`, `addGeometricShape`, `ShapeFill`). These are NOT CSS/DOM values.

---

## Spacing Scale (Slide Content)

All values in points. Based on a 12-point base unit for slide content (not 8px web scale).

| Token | Value | Usage |
|-------|-------|-------|
| slide-margin | 36 pt | Minimum distance from all slide edges to any content |
| region-gap | 24 pt | Gap between major layout regions (e.g., text column and chart area) |
| element-gap | 16 pt | Gap between elements within a region (e.g., title to body text) |
| inner-padding | 12 pt | Padding inside bordered elements (callout box, table cells) |
| bullet-indent | 18 pt | Left indent for bullet list items. Note: 18pt is 1.5x the 12pt base unit; not a multiple of 4 but a standard half-increment in the slide point grid. |
| table-cell-pad | 6 pt | Internal cell padding for table cells. Note: 6pt is 0.5x the 12pt base unit; not a multiple of 4 but a standard half-increment in the slide point grid. |

Exceptions: bullet-indent (18pt) and table-cell-pad (6pt) deviate from multiples-of-4 as documented above. Both are half-base and 1.5x-base multiples of the 12pt slide unit, which is the natural grid for point-based slide layout.

## Typography (Slide Content)

Font: **Calibri** (PowerPoint default, universally available, professional data presentation font per D-04).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Slide Title | 28 pt | Bold (700) | 1.2 (34 pt) | Main slide heading — one per slide |
| Key Insight | 22 pt | Bold (700) | 1.3 (29 pt) | Text inside the callout box (TEXT-03) |
| Body Text | 18 pt | Regular (400) | 1.5 (27 pt) | Bullet points, paragraphs, summaries |
| Table / Detail | 14 pt | Regular (400) for data, Bold (700) for headers | 1.2 (17 pt) | Table column headers (bold), table data cells (regular), chart placeholder label |

**Constraints:**
- Exactly 4 sizes this phase: 28, 22, 18, 14.
- Exactly 2 weights: Regular (400) and Bold (700). No other weights.
- All text uses Calibri. No font mixing.
- Title text: left-aligned. Body text: left-aligned. Table headers: center-aligned. Table data: right-aligned for numbers, left-aligned for text.

## Key Insight Callout Box (Detailed Spec)

The callout box is the most distinctive visual element — it must be unmissable (per D-05).

**Visual treatment:** Left-bordered accent box (inspired by blockquote styling).

| Property | Value |
|----------|-------|
| Shape type | Rectangle (addGeometricShape) |
| Fill | #EBF0FA (Light Summit Blue) |
| Left border | 3 pt, #0F1330 (Summit Navy) — achieved via a thin rectangle shape overlaid at the left edge |
| Other borders | None visible (fill blends with slight contrast to white slide) |
| Corner radius | 0 (sharp corners for professional/corporate look) |
| Text font | Calibri, 22 pt, Bold (700), #0F1330 |
| Text alignment | Left-aligned, vertically centered |
| Inner padding | 12 pt all sides (18 pt on left to clear the accent border) |

**Implementation note:** Office.js `addGeometricShape` does not support per-side border control. Implement the left accent border as a separate thin rectangle shape (width: 3pt, same height as callout box) positioned at the left edge of the callout box, filled with #0F1330. Group the two shapes if possible (addGroup, API 1.8).

## Interaction States (Slide Insertion)

### Test Trigger Buttons (Taskpane)

Fluent UI v9 `<Button>` components. Four buttons, one per layout template.

| State | Visual | Behavior |
|-------|--------|----------|
| Default | Fluent `appearance="secondary"` | Clickable |
| Hover | Fluent default hover (subtle background shift) | Cursor pointer |
| Loading | Fluent `appearance="secondary"` + `<Spinner>` replaces button text | During PowerPoint.run() execution; all buttons disabled |
| Success | Brief green checkmark icon next to button (2s, then revert to default) | Slide was inserted |
| Error | Red MessageBar below buttons with error text | PowerPoint.run() rejected |

### Slide Insertion Flow

1. User clicks test button
2. All 4 buttons become disabled + spinner on clicked button
3. `PowerPoint.run()` executes: add slide, add shapes, format content
4. On success: spinner stops, brief success indicator, buttons re-enabled
5. On error: spinner stops, error MessageBar shown, buttons re-enabled

| Requirement | Implementation |
