    Export `addTable(shapes: PowerPoint.ShapeCollection, headers: string[], rows: (string | number)[][], region: LayoutRegion): PowerPoint.Shape`:

    1. Build `values` 2D array: first row = headers, remaining rows = data with each cell run through `formatCellValue()` to get display strings.

       - Use `formatCellValue()` result's `isNumeric` flag to determine alignment per cell.

    3. Call `shapes.addTable(rows.length + 1, headers.length, { left, top, width, height, values, specificCellProperties })`.
       - DO NOT set `style` property (requires API 1.9, not available — Pitfall 3).
       - Return the shape reference.

  <action>
    **layoutEngine.ts** — slide-level operations (per D-01, D-03, LYOT-02, LYOT-03):

    1. Export `async function detectSlideWidth(context: PowerPoint.RequestContext): Promise<number>`:
       - Try to read `context.presentation.slides.getItemAt(0)` width if slides exist. Load properties, sync, return width.
       - If no slides exist or reading fails, return `960` (widescreen default).
       - Catches errors gracefully, always returns a number.

    2. Export `async function addSlideAtCurrentPosition(context: PowerPoint.RequestContext): Promise<PowerPoint.Slide>`:
       - **PREVIEW path (per D-03):** Runtime check `Office.context.requirements.isSetSupported("PowerPointApi", "1.9")` (index support). If supported, get selected slide index, call `context.presentation.slides.add({ index: selectedIndex + 1 })`.
       - **GA fallback:** Call `context.presentation.slides.add()` (appends to end).
       - After `context.sync()`, get the newly added slide reference via `slides.getItemAt()` (either at the inserted index or at `count - 1` for fallback).
       - Return the slide proxy object.

    **slideRenderer.ts** — top-level orchestrator (per LYOT-02):

    Export `async function insertSlide(content: SlideContent): Promise<void>`:

    Wraps everything in a single `PowerPoint.run(async (context) => { ... })` call (Pattern 1).

    Inside the run:
    1. `const slideWidth = await detectSlideWidth(context);`
    2. `const slide = await addSlideAtCurrentPosition(context);`
    3. `const shapes = slide.shapes;`
    4. Based on `content.type`, use the correct template from TEMPLATES, adapt each region with `adaptLayout(region, slideWidth)`, then call the appropriate renderer functions:

    - `"text-only"`: addTitle, addBody (bullets), addCalloutBox (insight)
    - `"chart-text"`: addTitle, addChartPlaceholder (chart region), addBody (summaryBullets in text region), addCalloutBox (insight in callout region)
    - `"table-text"`: addTitle, addTable (headers + rows in table region), addSummaryText (summary in summary region — plain text, not bulleted)
    - `"full-combination"`: addTitle, addChartPlaceholder (chart region), addTable (in table region), addCalloutBox (insight in callout region)

    5. Final `await context.sync()` to commit all shapes.

    Import from `../slide/textRenderer`, `../slide/tableRenderer`, `../slide/placeholderRenderer`, `../slide/layoutEngine`, `../slide/constants`, `../slide/types`.
  </action>
