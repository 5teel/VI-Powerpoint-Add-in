# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Fluent UI v9 provides the design system) |
| Preset | not applicable |
| Component library | @fluentui/react-components 9.73.4 |
| Icon library | @fluentui/react-icons (bundled with Fluent UI v9) |
| Font | Segoe UI (Fluent UI v9 default, already loaded in Office context) |

**Note:** Fluent UI v9 provides its own design tokens via `tokens` export. All spacing, typography, and color values below map to Fluent UI v9 token names where applicable. Custom values are used only for Summit VI branding.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Fluent Token | Usage |
|-------|-------|-------------|-------|
| xs | 4px | spacingHorizontalXS | Icon gaps, inline padding |
| sm | 8px | spacingHorizontalS | Compact element spacing, input padding |
| md | 16px | spacingHorizontalM | Default element spacing, card padding |
| lg | 24px | spacingHorizontalL | Section padding, header padding |
| xl | 32px | spacingHorizontalXL | Layout gaps |
| 2xl | 48px | spacingHorizontalXXL | Major section breaks |

Exceptions: Touch target minimum 44px for the send button (accessibility requirement in narrow taskpane). Office taskpane width is 320-400px; all layout must work at 320px minimum.

## Layout Contract

The taskpane uses a single-column vertical layout with three regions. This maps to decision D-01 (chat-style layout).

**Primary focal point:** The input area (text input + send button) at the bottom of the taskpane. This is where the user's attention starts and returns after each interaction. The response area above serves as a feedback surface that draws the eye only when content appears.

```
+---------------------------+
|  HEADER (fixed, 48px)     |
|  [Logo] Summit VI         |
+---------------------------+
|                           |
|  RESPONSE AREA (flex: 1)  |
|  Scrollable vertical      |
|  - Connectivity status    |
|  - API response display   |
|  - Empty state            |
|                           |
+---------------------------+
|  INPUT AREA (fixed)       |  <-- primary focal point
|  [Text input] [Send btn]  |
+---------------------------+
```

| Region | Height | Behavior |
|--------|--------|----------|
| Header | 48px fixed | Always visible. Summit logo (24px height) + "Summit VI" brand text. Background: `#0F1330`. |
| Response area | flex: 1 (fills remaining space) | Scrollable overflow-y: auto. Contains connectivity test results and API responses. |
| Input area | auto (approx 56px) | Fixed to bottom. Contains text input + send button on a single row. 8px padding all sides. |

**Minimum taskpane width:** 320px. All components must be usable at this width.

## Interaction States

### Send Button

| State | Visual | Behavior |
|-------|--------|----------|
| Default | `#0F1330` background, white text "Send Query" | Clickable |
| Hover | `#1A2040` background (10% lighter) | Cursor: pointer |
| Disabled | Fluent UI disabled tokens (grayed out) | During API call, while input is empty |
| Loading | Hidden; replaced by Spinner in the response area | Button remains enabled but shows spinner above |

### Text Input

| State | Visual | Behavior |
|-------|--------|----------|
| Empty | Placeholder text visible: "Ask a question to test the connection..." | Send button disabled |
| Focused | Fluent UI focus ring (2px outline, brand color) | Standard keyboard interaction |
| Filled | User text visible, placeholder hidden | Send button enabled |
| Submitting | Input disabled, text preserved | Re-enabled after response or error |

### Connectivity Test Display

| State | Visual | Behavior |
|-------|--------|----------|
| Idle (empty) | Empty state copy centered in response area | Waiting for user action |
| Loading | Spinner + "Connecting to Cube AI..." text | Input disabled |
| Success | Green Badge + success copy + response time + raw response preview (truncated to 500 chars) | Input re-enabled |
| Error | Red MessageBar with specific error copy from copywriting contract | Input re-enabled, user can retry |

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Keyboard navigation | Tab order: Input -> Send button. Enter in input submits. Fluent UI provides focus management. |
| Screen reader | Spinner has `label="Connecting to Cube AI"`. Success/error states use `role="status"` live region. |
| Color contrast | White text on `#0F1330` = 15.5:1 ratio (passes WCAG AAA). Body text `#242424` on `#FFFFFF` = 15.1:1 (passes WCAG AAA). |
| Touch targets | Send button minimum 44x32px. Input field full-width minus button. |
| Motion | Spinner is the only animated element. Respects `prefers-reduced-motion` via Fluent UI defaults. |
