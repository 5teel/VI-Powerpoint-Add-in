# Phase 4.1: Guided Slide Builder - Context

**Gathered:** 2026-03-24
**Status:** Ready for planning

<domain>
## Phase Boundary

Build a step-by-step wizard as a new tab ("Build Slide") alongside the existing Ask Summit chat. The wizard guides users through 4 linear steps — Brand → Image → Purpose → Review — to produce polished, branded slides with product imagery. Session-based image storage using base64 in React state.

</domain>

<decisions>
## Implementation Decisions

### Wizard Flow
- **D-01:** New "Build Slide" tab alongside existing "Ask Summit" and "Slide Options" tabs. Does not replace the chat — both remain available.
- **D-02:** 4 linear steps with back navigation: Brand → Image → Purpose → Review. Users can go back but not skip ahead. Simple progress indicator at top showing current step.
- **D-03:** The wizard produces a structured prompt combining brand context, analysis purpose, and image reference, then calls the same `streamCubeAI` + `extractSlideContent` + `insertSlide` pipeline as the chat flow.

### Image Upload & Storage
- **D-04:** File picker (standard `<input type="file">` button) for image upload. PNG and JPG only.
- **D-05:** Max file size 6MB. Resize to max 800px on longest side for slide quality. Store as base64 in React state (session-only, not persisted).
- **D-06:** Show a thumbnail preview of the uploaded image in the wizard after upload. User can replace it by uploading again.
- **D-07:** The uploaded image is inserted into the created slide via `ShapeFill.setImage(base64)` on a geometric shape in the designated image region.

### Brand Context
- **D-08:** Free text input for brand name — user types "Coca Cola", "Red Bull", etc. No predefined dropdown.
- **D-09:** The brand name is included in the Cube AI prompt to focus the analysis (e.g., "Provide a performance breakdown for [brand]").

### Slide Preview & Approval
- **D-10:** Inputs-only preview card showing brand name, analysis purpose, and uploaded image thumbnail. User clicks "Build Slide" to confirm. Cube AI is called only after approval — no pre-build API call. (Revised from original per RESEARCH.md recommendation — showing AI-generated content would require a wasted API call before approval.)
- **D-11:** After "Build Slide" is clicked, the slide is created in PowerPoint and a success confirmation is shown in the wizard.

### Claude's Discretion
- Progress indicator design (dots, numbered steps, progress bar)
- Wizard component architecture (single component with step state, or separate step components)
- Analysis purpose step UI (free text, predefined options, or both)
- How the image region is sized and positioned on the slide layout
- How the brand name and purpose feed into the system prompt alongside the schema

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- **placeholderRenderer.ts**: Already uses `ShapeFill.setImage(base64)` — same API for product images
- **promptBuilder.ts**: Can be extended or a new `buildGuidedPrompt()` created that includes brand + purpose context
- **App.tsx**: Tab routing already works — adding a third tab is straightforward
- **ChatPanel.tsx**: "Create Slide" button lifecycle pattern (idle → creating → created → failed) can be reused in the wizard

### Established Patterns
- React 18 + Fluent UI v9 components
- Callback-based streaming with `streamCubeAI()`
- Schema JSON extraction → `SlideContent` → `insertSlide()`
- TypeScript strict mode

### Integration Points
- New `WizardPanel` component added as a tab in `App.tsx`
- Wizard calls the same pipeline: `buildGuidedPrompt()` → `streamCubeAI()` → `extractSlideContent()` → `insertSlide()`
- Image stored in React state, passed to `insertSlide()` which uses `ShapeFill.setImage(base64)`
- `SlideContent` types may need an optional `productImageBase64` field

</code_context>

<specifics>
## Specific Ideas

- The wizard should feel clean and modern — consistent with the subtle blue gradient background and rounded edges already in the app
- The image upload step should show a clear preview of the uploaded product image so the user knows what will appear on the slide
- The review step should make it obvious what's about to be built — brand name, analysis focus, image preview, all in one card
