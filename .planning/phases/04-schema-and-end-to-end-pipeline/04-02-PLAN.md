    **2. Extend ChatMessage interface** (per D-05, D-06, Pitfall 6):
    Add `slideState` field to track per-message slide creation:
    ```typescript
    interface ChatMessage {
      role: "user" | "assistant" | "error";
      content: string;
      error?: CubeAIError;
      slideState?: "idle" | "creating" | "created" | "failed";
    }
    ```

    **3. Wrap user question with system prompt** (per D-01, D-02, TASK-01):
    In `handleSubmit`, wrap the question before passing to `streamCubeAI`:
    ```typescript
    const wrappedQuestion = buildSlidePrompt(question);
    const controller = streamCubeAI(wrappedQuestion, chatId, { ... });
    ```
    IMPORTANT per Pitfall 5: The `messages` state still stores the original `question` as the user message content (not the wrapped version). The wrapping is transparent to the UI.

    **4. Set slideState on completed assistant messages:**
    In the `onComplete` callback, set `slideState: "idle"` on the new assistant message so the Create Slide button appears:
    ```typescript
    onComplete: (result: CubeAIStreamResult) => {
      setMessages((prev) => [...prev, {
        role: "assistant",
        content: result.content || "(No response received)",
        slideState: "idle",
      }]);
      // ... rest unchanged
    },
    ```

    **5. Add handleCreateSlide callback** (per D-05, D-06, Pitfall 6):
    ```typescript
    const handleCreateSlide = useCallback(async (messageIndex: number) => {
      const msg = messages[messageIndex];
      if (!msg || msg.role !== "assistant" || msg.slideState !== "idle") return;

      setMessages(prev => prev.map((m, i) =>
        i === messageIndex ? { ...m, slideState: "creating" } : m
      ));

      try {
        const content = extractSlideContent(msg.content) ?? fallbackToTextOnly(msg.content);
        await insertSlide(content);
        setMessages(prev => prev.map((m, i) =>
          i === messageIndex ? { ...m, slideState: "created" } : m
        ));
      } catch (err) {
        setMessages(prev => prev.map((m, i) =>
          i === messageIndex ? { ...m, slideState: "failed" } : m
        ));
      }
    }, [messages]);
    ```

    **6. Render Create Slide button and confirmation** in assistant message block:
    After the existing assistant message text div, add:
    - When `slideState === "idle"`: A Fluent UI `<Button>` with text "Create Slide", appearance="primary", styled with Summit Navy background (`#0F1330`)
    - When `slideState === "creating"`: A disabled button with `<Spinner size="tiny" />` and text "Creating..."
    - When `slideState === "created"`: A `<Text>` element showing "Slide created" in green (#107C10) with a checkmark
    - When `slideState === "failed"`: A `<Text>` showing "Failed to create slide" in red, plus a retry button

    Button placement: Below the assistant message bubble, left-aligned, with 4px top margin.

    **7. Add `messages` to handleCreateSlide dependency array** in useCallback.
