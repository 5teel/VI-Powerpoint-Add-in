**Exported function:**

```typescript
export function streamCubeAI(
  question: string,
  chatId: string | null,
  callbacks: StreamCallbacks
): AbortController
```

**Implementation requirements:**

1. **Immediately** call `callbacks.onPhaseChange("connecting")` and create AbortController + timeout (per D-08).

2. **Request construction** (per D-01, D-02):
   - POST to `CUBEAI_CONFIG.baseUrl`
   - Headers: `Content-Type: application/json`, `Authorization: Api-Key ${CUBEAI_CONFIG.apiKey}`
   - Body: `{ input: question, sessionSettings: { externalId: CUBEAI_CONFIG.externalId, ...(chatId ? { chatId } : {}) } }`
   - Signal: `controller.signal`

3. **Error handling on fetch failure** (preserve existing error classification):
   - TypeError with "Failed to fetch" or "NetworkError" -> type "cors", message "Unable to reach Cube AI. Please check your connection and try again.", retryable false
   - AbortError -> type "timeout", message "The request timed out. Cube AI may be busy -- please try again.", retryable true
   - Other errors -> type "network", message "Connection failed. Please check your network and try again.", retryable true
   - Per D-06: NO HTTP status codes in user-facing messages. Log technical details with `console.error("Cube AI error:", err)`.

4. **HTTP error handling:**
   - 401/403 -> type "auth", message "Authentication failed. The API key may be invalid.", retryable false
   - 429 -> type "server", message "Cube AI is busy. Please wait a moment and try again.", retryable true
   - 5xx -> type "server", message "Cube AI encountered an error. Please try again.", retryable true
   - Other non-ok -> type "unknown", message "Something went wrong. Please try again.", retryable false
   - Null response body -> type "empty", message "Cube AI returned an empty response. Please try again.", retryable true

5. **On successful HTTP response:** call `callbacks.onPhaseChange("connected")`.

6. **NDJSON streaming loop** (per D-03, 200ms throttle):
   - Use `response.body.getReader()` and `new TextDecoder()`
   - Read chunks in `while(true)` loop with `{ done, value } = await reader.read()`
   - Decode with `decoder.decode(value, { stream: true })` (Pitfall 1)
   - Buffer splitting: `buffer += decoded; const lines = buffer.split("\n"); buffer = lines.pop() || "";`
   - Parse each non-empty line as JSON:
     - `message.state?.chatId` -> capture as responseChatId
     - `message.role === "assistant" && message.content && message.isDelta` -> `streamContent += message.content`
     - `message.role === "assistant" && message.content && !message.isDelta && !message.isInProcess` -> `streamContent = message.content` (final replaces accumulated)
   - On first delta content, call `callbacks.onPhaseChange("streaming")` (track with `firstDelta` boolean)
   - **Throttled flush** (per D-03): Track `lastFlush = performance.now()`, flush via `callbacks.onContent(streamContent)` when `now - lastFlush >= 200`

7. **After read loop exits** (done === true):
   - Process remaining buffer content (Pitfall 3)
   - Always perform final un-throttled flush: `callbacks.onContent(streamContent)` (Pitfall 4)
   - Call `callbacks.onPhaseChange("complete")`
   - Call `callbacks.onComplete({ content: streamContent, chatId: responseChatId })`

8. **Cleanup:** Use `try/finally` to always `clearTimeout(timeout)` (Pitfall 5).

9. **Abort handling** (Pitfall 2): If `controller.signal.aborted` is true when an error occurs, do NOT call `onError` -- the caller intentionally aborted.

10. **Return** the AbortController so the caller can cancel the request.

The entire async logic runs inside an immediately-invoked async arrow function `(async () => { ... })()` so `streamCubeAI` itself is synchronous and returns the AbortController immediately.
