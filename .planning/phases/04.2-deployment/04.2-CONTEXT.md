# Phase 4.2: Deployment - Context

**Gathered:** 2026-03-30
**Status:** Ready for planning

<domain>
## Phase Boundary

Deploy the add-in's production build to Railway via Docker, host source code on GitHub, and update the manifest so the add-in loads from a remote HTTPS URL instead of localhost:3100. This enables distribution to users with Summit email accounts (sideload or future M365 admin center rollout).

</domain>

<decisions>
## Implementation Decisions

### API Key Security
- **D-01:** Inject Cube AI API key via webpack `DefinePlugin` at build time using Railway environment variables. Remove the hardcoded key from `config.ts` and replace with `process.env.CUBEAI_API_KEY` (or equivalent define).
- **D-02:** The key will still be in the client-side bundle (visible in DevTools) — acceptable for internal-only use. No proxy server needed at this stage.

### GitHub Repository
- **D-03:** Private repository under the user's personal GitHub account. Can be transferred to an org later.
- **D-04:** `.planning/` directory added to `.gitignore` — planning artifacts stay local only.
- **D-05:** `config.ts.txt` (contains raw secrets) must be gitignored. Verify `.env` is already gitignored (it is).

### Railway Configuration
- **D-06:** Docker container deployment. Dockerfile should: install deps, run production webpack build, serve `dist/` folder with a lightweight HTTP server (e.g., nginx or serve).
- **D-07:** Railway auto-generated subdomain with free HTTPS (e.g., `summit-vi.up.railway.app`). No custom domain needed.
- **D-08:** Railway environment variables for: `CUBEAI_API_KEY`, `CUBEAI_BASE_URL`, `CUBEAI_EXTERNAL_ID`, `CUBEAI_TIMEOUT_MS`.

### Manifest Strategy
- **D-09:** Single `manifest.xml` — webpack's existing `CopyWebpackPlugin` replaces `localhost:3100` with the Railway URL at production build time. Update `urlProd` in `webpack.config.js` to the Railway subdomain.
- **D-10:** The production `manifest.xml` in `dist/` will have all Railway URLs. This is the file distributed for sideloading or M365 admin upload.

### Claude's Discretion
- Dockerfile specifics (base image, multi-stage build, server choice)
- webpack `DefinePlugin` variable naming convention
- Any `.dockerignore` contents
- Railway deployment configuration (railway.toml if needed)

</decisions>
