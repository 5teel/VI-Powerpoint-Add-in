## API Key Security

| Option | Description | Selected |
|--------|-------------|----------|
| Env var at build time | Inject via webpack DefinePlugin + Railway env vars. Key is in the bundle but not in source code. Git-safe. | ✓ |
| Keep hardcoded for now | Internal-only tool, acceptable risk. Just ensure config.ts.txt doesn't go to GitHub. | |
| Proxy server | Route API calls through a backend that holds the key. Most secure but adds infrastructure. | |

**User's choice:** Env var at build time (Recommended)
**Notes:** Key still visible in client bundle — acceptable for internal use.

## Railway Config

| Option | Description | Selected |
|--------|-------------|----------|
| Static site via Nixpacks | Railway auto-detects Node, runs npm run build, serves dist/. Zero config. | |
| Express static server | Add minimal server.js that serves dist/. More control but more to maintain. | |
| Docker container | Full control via Dockerfile. Flexible. | ✓ |

**User's choice:** Docker container

### Follow-up: Domain

| Option | Description | Selected |
