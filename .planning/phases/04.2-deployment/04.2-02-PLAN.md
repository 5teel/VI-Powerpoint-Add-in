**Get the deployment URL:**

Once deployed, Railway assigns a URL like `https://summit-vi-production.up.railway.app`.

**If the Railway URL is NOT exactly `summit-vi.up.railway.app`:**

The actual Railway subdomain may differ from what was assumed. If so, Claude must:
1. Update `webpack.config.js` `urlProd` to the actual Railway URL (with trailing slash)
2. Update `manifest.xml` `AppDomains` and `SupportUrl` to the actual Railway URL
3. Commit, push, and wait for Railway to redeploy

This is expected — Railway subdomain names are auto-assigned per-project.
